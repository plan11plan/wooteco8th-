## **예외 처리 정책**

### **원칙**

- **InputView**: 검증 없이 문자열만 반환
- **VO (VisitDate, Menus)**: 모든 검증 수행 (파싱 + 도메인)
- **Controller**: 예외를 catch하여 에러 메시지 출력 및 재입력 처리

### **예외 메시지**

- 날짜 오류: `[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요.`
- 주문 오류: `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 모든 예외```는 `IllegalArgumentException` 사용
```

### 1. 입출력 기본 구조
- 처음 안내 메시지 출력
- 방문 날짜 입력
- 주문 메뉴 입력
- 혜택 안내 메시지 출력

### 2. 도메인 모델
- 입력 받은 날짜 파싱 및 검증 모델 구현
- 입력 받은 메뉴 파싱 및 검증 모델 구현


### 3. 계산
할인과 증정은 중복 가능하다.

- 크리스마스 디데이 할인
  - 기간
    - 12월 25일
  - 보상
    - 1,000원으로 시작하여 크리스마스가 다가올수록 할인 금액이 100원씩 증가
    - 총주문 금액에서 할인
    - (e.g. 시작일인 12월 1일에 1,000원, 2일에 1,100원, ..., 25일엔 3,400원 할인)
- 평일 할인
  - 기간
    - 2023.12.1 ~ 2023.12.31
    -  (일요일~목요일)
  - 보상
    - 디저트메뉴 1개당 2,023원 할인
- 주말 할인
  - 기간
    - 2023.12.1 ~ 2023.12.31
    - (금요일, 토요일)
  - 보상
    - 메인 메뉴 1개당 2,023원 할인
- 특별 할인
  - 기간
    - 2023.12.1 ~ 2023.12.31
    - 12월달의 모든 일요일, 25일
  - 보상
    - 총 주문금액에서 1,000원 할인
- 증정 이벤트
  - 기간
    - 2023.12.1 ~ 2023.12.31
  - 조건
    - 할인 전 총주문 금액이 12만원 이상
  - 보상
    - 샴페인 1개 증정
### 4. 통합 및 출력
(없으면 -> 없음), (0원이면 -> 0원)

혜택 받은 금액은 -로 표시한다.
- 주문 메뉴
- 할인 전 총주문 금액 
- 증정 메뉴
- 혜택 내역
- 총혜택 금액
- 할인 후 결제 금액
- 12월 이벤트 배지

---
## 정보

총 혜택 금액에 따른 뱃지 증정
- 5천 원 이상: 별
- 1만 원 이상: 트리
- 2만 원 이상: 산타

출력 순서는 자유롭다. 

총혜택 금액 = 할인 금액의 합계 + 증정 메뉴의 가격

할인 후 예상 결제 금액 = 할인 전 총주문 금액 - 할인 금액

### 제약
- 총주문 금액 10,000원 이상부터 이벤트가 적용됩니다.
- 음료만 주문 시, 주문할 수 없습니다.
- 메뉴는 한 번에 최대 20개까지만 주문할 수 있습니다.
  - (e.g. 시저샐러드-1, 티본스테이크-1, 크리스마스파스타-1, 제로콜라-3, 아이스크림-1의 총개수는 7개)
